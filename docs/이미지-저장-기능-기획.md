# 결과지 이미지 저장 기능 기획

| 항목 | 내용 |
| --- | --- |
| 문서 상태 | 기획 초안 |
| 작성일 | 2026-07-01 |
| 관련 화면 | `/complete`, `PdfPreviewOverlay`, `PrintableForm` |
| 관련 문서 | `docs/PRD.md`, `docs/PDF-전송-설계.md` |

## 1. 목적

완료 화면에서 보호자가 작성 결과지를 PDF뿐 아니라 **이미지 파일(PNG)**로 저장할 수 있게 한다.

한국 사용자는 카카오톡 인앱 브라우저에서 모바일 웹을 여는 경우가 많다. 이 환경에서는 PDF 다운로드나 파일 공유가 기기·OS·브라우저별로 일관되지 않을 수 있으므로, **이미지 미리보기 + 길게 눌러 저장** 흐름을 1차 UX로 제공한다.

## 2. 사용자 문제

- 보호자는 PDF 저장 위치를 찾기 어렵거나, 카카오톡 인앱 브라우저에서 다운로드 동작을 이해하기 어렵다.
- 카카오톡 대화방으로 공유하려면 사진첩/갤러리에 저장된 이미지가 PDF보다 익숙하다.
- 센터 비치 기기에서 작성하는 경우, 보호자가 결과 사본을 쉽게 가져갈 수 있어야 한다.

## 3. 1차 범위

### 포함

- 완료 화면에 **결과 이미지 저장** 버튼 추가.
- 작성 결과지를 PNG 이미지로 생성.
- 생성된 이미지를 화면에 크게 미리보기로 표시.
- 카카오톡 인앱 브라우저 기준 안내 문구 제공.
  - "이미지를 길게 눌러 저장하세요."
  - "저장 후 카카오톡 대화방에서 사진으로 보낼 수 있어요."
- 지원 브라우저에서는 다운로드 버튼도 함께 제공.
- 이미지 생성 실패 시 PDF 저장/인쇄로 fallback.

### 제외

- 서버 업로드.
- DB 저장.
- 카카오톡 특정 대화방으로 자동 전송.
- 카카오 공유 SDK 연동.
- 이미지 파일을 센터 서버에 보관.

## 4. 권장 사용자 흐름

```
[작성 완료]
   ↓
[결과 이미지 저장] 버튼 선택
   ↓
[이미지 생성 중]
   ↓
[이미지 미리보기 오버레이]
   ├─ 이미지를 길게 눌러 저장
   ├─ 다운로드 버튼으로 저장
   └─ 닫기
```

완료 화면의 주요 버튼 우선순위:

1. 결과 이미지 저장
2. 문서 보기 · PDF 저장
3. 처음으로 (새 작성)

## 5. UI 문구

### 완료 화면

버튼:

- `문서 보기 · PDF 저장`

### 이미지 생성 중

### 이미지 미리보기

- `닫기`

실패 메시지:

## 6. 결과 이미지 형태

### 1차 결정: 긴 단일 PNG

1차 구현은 **전체 결과지를 하나의 긴 PNG 이미지**로 생성한다.

이유:

- 보호자가 여러 장을 저장하지 않아도 된다.
- 카카오톡 대화방에 사진 1장으로 공유할 수 있다.
- 현재 `PrintableForm`이 전체 결과지를 한 컴포넌트로 렌더링하고 있어 구현 범위가 작다.

주의:

- 모든 상세 영역이 선택된 경우 이미지 높이가 길어질 수 있다.
- 일부 모바일 브라우저는 매우 큰 캔버스 생성에 실패할 수 있다.
- 실패 시 PDF 저장을 안내한다.

### 향후 개선: 페이지 단위 PNG

긴 이미지 생성 실패율이 높거나 가독성이 떨어지면, A4 페이지 단위로 여러 PNG를 생성하는 v2를 검토한다.

## 7. 기술 설계

### 렌더링 소스

현재 인쇄/PDF 미리보기에 쓰는 `PrintableForm`을 이미지 생성에도 재사용한다.

- 문항과 응답 표시 로직을 중복 구현하지 않는다.
- PDF와 이미지 결과물의 내용 불일치를 피한다.
- 기존 PDF 미리보기 스타일을 기준으로 이미지 크기와 여백을 맞춘다.

### 구현 방식

브라우저에서 DOM을 이미지로 변환한다.

권장 라이브러리:

- `html-to-image`

예상 구조:

- `components/report-image-overlay.tsx`
  - 이미지 생성 상태 관리.
  - 생성된 `Blob`/object URL 미리보기.
  - 다운로드 버튼.
  - 길게 눌러 저장 안내.
- `lib/report-image.ts`
  - DOM 노드를 PNG Blob으로 변환하는 순수 유틸.
  - 파일명 생성.

### 생성 절차

1. 화면 밖 또는 오버레이 내부에 이미지 생성용 `PrintableForm`을 렌더링한다.
2. 고정 폭 컨테이너를 사용한다.
   - 권장 폭: `800px`
   - 배경: 흰색
   - 패딩: 기존 PDF 미리보기와 동일한 `24px`
3. `html-to-image`로 PNG Blob을 생성한다.
4. `URL.createObjectURL(blob)`으로 미리보기 이미지를 표시한다.
5. 다운로드 버튼은 `<a download>` 또는 동적 anchor 클릭으로 처리한다.
6. 오버레이가 닫히면 object URL을 해제한다.

### 파일명

형식:

```
발달체크리스트_{아동이름}_{YYYY-MM-DD}.png
```

이름이 없으면:

```
발달체크리스트_{YYYY-MM-DD}.png
```

파일명에 사용할 수 없는 문자는 제거한다.

## 8. 개인정보·보안

- 이미지 생성은 브라우저 내부에서만 수행한다.
- 서버 업로드를 하지 않는다.
- 이미지 Blob은 현재 브라우저 세션에서만 유지한다.
- 오버레이를 닫으면 object URL을 해제한다.
- `처음으로 (새 작성)` 실행 시 기존 폼 초기화 정책을 그대로 따른다.

중요:

- 보호자가 이미지를 저장하면 해당 이미지는 보호자 또는 센터 기기의 사진첩/파일 앱에 남을 수 있다.
- 센터 비치 기기에서 보호자에게 결과지를 전달하는 운영 방식이라면, 저장 후 기기에 남은 이미지 삭제 안내 또는 초기화 운영 정책이 필요하다.

## 9. 접근성·모바일 UX

- 버튼 터치 영역은 최소 44px 이상으로 유지한다.
- 생성 중 상태는 `aria-live`로 알린다.
- 실패 시 같은 화면에서 PDF 저장으로 이동할 수 있게 한다.
- 이미지 미리보기는 pinch zoom이 가능하도록 브라우저 기본 이미지 동작을 막지 않는다.
- 오버레이 열림 시 닫기 또는 다운로드 버튼으로 포커스를 이동한다.

## 10. 완료 조건

- 완료 화면에서 `결과 이미지 저장` 버튼이 보인다.
- 버튼 선택 시 결과 이미지가 생성된다.
- 생성된 이미지는 화면에서 확인할 수 있다.
- 카카오톡 인앱 브라우저에서 이미지를 길게 눌러 저장할 수 있다.
- 지원 브라우저에서 `이미지 다운로드` 버튼으로 PNG 저장이 가능하다.
- 이미지 내용은 PDF 미리보기와 동일한 기본정보, 02·03·04 응답, 추가 기록을 포함한다.
- 이미지 생성 실패 시 사용자에게 실패 메시지와 PDF 저장 대안을 보여준다.
- `pnpm build`가 통과한다.

## 11. 검증 계획

### 로컬

- 일반 브라우저에서 이미지 생성 확인.
- 다운로드 파일명 확인.
- 기본정보/체크 표시/한글 깨짐 여부 확인.
- 선택된 04 상세 영역과 미선별 영역 표기 확인.
- 모든 04 영역이 선택된 긴 결과지에서 생성 성공 여부 확인.

### 실제 기기

- iPhone Safari.
- Android Chrome.
- iOS 카카오톡 인앱 브라우저.
- Android 카카오톡 인앱 브라우저.

확인 항목:

- 버튼 터치 가능성.
- 이미지 생성 시간.
- 이미지 길게 눌러 저장 가능 여부.
- 저장된 이미지가 갤러리/사진 앱 또는 파일 앱에서 열리는지.
- 카카오톡 대화방에 사진으로 첨부 가능한지.

## 12. 미결정 질문

1. 결과 이미지는 **전체 결과지 1장**으로 충분한가, 아니면 처음부터 **A4 페이지 단위 여러 장**이 필요한가?
   - 권장: 1차는 전체 결과지 1장.
2. 센터 비치 기기에서 보호자가 이미지를 저장하는 운영을 허용할 것인가?
   - 허용 시 기기 사진첩/다운로드 폴더에 개인정보가 남을 수 있다.
3. 이미지에 포함할 범위는 PDF와 완전히 동일하게 할 것인가?
   - 권장: 1차는 PDF 미리보기와 동일.
4. 완료 화면의 1순위 CTA를 `결과 이미지 저장`으로 바꿀 것인가?
   - 권장: 카카오톡 사용자를 고려해 이미지 저장을 1순위로 배치.
