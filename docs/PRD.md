## 1. 개요

아동발달센터에 방문한 보호자가 **센터에 비치된 기기(태블릿/폰)**에서 초기상담 체크리스트를 직접 작성하는 모바일 웹 애플리케이션이다. 기존 종이 양식을 디지털화하여, 작성 완료 시 **PDF로 출력**하고 **카카오톡으로 공유**할 수 있다.

핵심 특징:
- 모바일 우선(mobile-first) 단일 페이지 설문
- 03번 빠른 선별 결과에 따라 04번 상세 문항을 **조건부로 노출**하여 작성 부담 최소화
- 별도 회원가입·로그인 없음 (현장 비치 기기)
- 서버 DB 영속 저장은 1차 범위에서 제외, **PDF 생성 + 카카오톡 공유**가 결과 산출물

---

## 2. 배경 및 문제 정의

- 현재 초기상담 체크리스트는 종이 양식으로 운영된다(`예/아니요 현장검사용`).
- 종이 양식의 한계:
  - 보호자가 모든 영역(언어·대근육·소근육·사회성·인지·감각예민·감각둔감)을 빠짐없이 채워야 해 작성 부담이 크고, 해당 없는 영역까지 훑어야 한다.
  - 작성 후 보관·전달·집계가 수기로 이루어져 누락·분실 위험이 있다.
  - 보호자가 작성 사본을 갖기 어렵다.
- 모바일 웹으로 전환하면:
  - 03번 선별을 통해 **관심 영역만 상세 작성**하도록 분기 가능
  - 작성 즉시 PDF 산출물 확보, 카카오톡으로 보호자/담당자에게 전달 가능
  - 입력 검증으로 누락 방지

---

## 3. 목표 및 성공 지표

### 3.1 목표
- 보호자가 **5분 이내**에 안내 없이 작성 완료할 수 있다.
- 종이 양식과 **동일한 문항·구조**를 유지하여 상담사가 결과를 그대로 활용할 수 있다.
- 작성 결과를 신뢰성 있게 PDF로 산출하고 공유할 수 있다.

### 3.2 성공 지표(예시)
- 작성 시작 → 완료 전환율 ≥ 90%
- 평균 작성 시간 ≤ 5분
- 필수 항목 누락으로 인한 제출 실패 후 재작성 이탈률 최소화
- PDF 생성 성공률 ≥ 99%

## 4. 사용자 및 페르소나

| 페르소나 | 설명 | 핵심 니즈 |
| --- | --- | --- |
| 보호자(주 사용자) | 센터 방문, 디지털 익숙도 다양, 어린 자녀 동반으로 주의 분산 | 짧고 명확한 작성, 큰 터치 영역, 쉬운 진행 |
| 상담사/검사자 | 작성 결과를 상담·검사 방향 설정에 활용 | 종이와 동일한 항목, 깔끔한 PDF, 빠른 전달 |
| 센터 운영자 | 비치 기기 관리 | 작성 후 자동 초기화, 개인정보 잔존 없음 |

## 5. 사용 시나리오 (User Flow)

```
[랜딩 화면]  ── 서비스 안내 + "검사 시작" 버튼
        ↓
[기본정보 입력]  ── 아동 이름·성별·생년월일(연령 자동계산), 보호자 이름·연락처
        ↓
[02. 초기 상담 기본 확인]  ── 예/아니요 6문항 (항상 표시)
        ↓
[03. 주호소 영역 빠른 선별]  ── 예/아니요 9문항 (항상 표시, 분기 트리거)
        ↓
[04. 상세 체크]  ── 03에서 "예" 체크된 영역의 상세 섹션만 동적 노출
        ↓
[검토/요약 화면]  ── 입력값 확인, "예" 응답 요약, 수정 가능
        ↓
[완료]  ── PDF 미리보기 → ① PDF 저장/인쇄  ② 카카오톡 공유
        ↓
[초기화]  ── 다음 보호자를 위해 입력값 전체 삭제
```

## 6. 기능 요구사항 (Functional Requirements)

### FR-0. 랜딩 화면
- 서비스 소개(아동발달센터 초기상담 체크리스트)와 간단한 작성 안내를 제공한다.
- "검사 시작" CTA 버튼으로 기본정보 입력 화면으로 진입한다.
- 개인정보 수집·이용 안내 문구 노출 여부는 §11(D-7)에서 결정.

### FR-1. 설문 화면 구성
- 02·03·04 설문은 원본 `docs/아동 발달 초기 체크리스트.md`의 구조와 문항을 **그대로** 따른다.
- 섹션 단위로 화면을 구성한다. (스텝형 다단계 또는 섹션별 스크롤 — §11 결정 필요)
- 진행률 표시(progress indicator)를 제공한다.

### FR-2. 기본정보 입력 (간소화)
- 보호자가 입력하는 항목은 다음 5개로 한정한다. (기존 종이 양식의 나머지 필드는 제외)

| 필드 | 입력 유형 | 비고 |
| --- | --- | --- |
| 아동 이름 | 텍스트 | 필수 |
| 아동 성별 | 선택(남/여) | 필수 |
| 아동 생년월일 | 날짜 선택 | 필수, 입력 시 현재 연령 자동 계산 |
| 보호자 이름 | 텍스트 | 필수 |
| 보호자 연락처 | 전화번호 | 필수, 형식 검증 |

- **현재 연령(만 N세 M개월)**: 생년월일 기반으로 자동 계산하여 화면에 표시하고 PDF·결과에 포함한다(읽기 전용).
- **작성 일시**: 작성 시작/완료 시각을 자동 기록하여 PDF·결과에 포함한다(사용자 입력 불필요).
- 제외 필드: 소속기관, 학년/반, 관계, 주 양육자, 형제자매, 상담자, 검사자.

### FR-3. 02. 초기 상담 기본 확인 (항상 표시)
- 예/아니요 6문항. 각 문항 단일 선택(예 또는 아니요).
- 분기에 영향 없음(기록 목적).

### FR-4. 03. 주호소 영역 빠른 선별 (항상 표시 · 분기 트리거)
- 예/아니요 9문항.
- 이 섹션의 "예" 응답이 04번 상세 섹션 노출을 결정한다(FR-5).

### FR-5. 04. 영역별 상세 체크 — 조건부 분기 ★핵심
- 03번 선별 문항과 04번 상세 섹션의 매핑:

| 03번 선별 문항 | 노출되는 04 상세 섹션 |
| --- | --- |
| 언어발달 지연이 의심된다 | 04-1. 언어발달 (8문항) |
| 대근육 발달 지연이 의심된다 | 04-2. 대근육 발달 (8문항) |
| 소근육 발달 지연이 의심된다 | 04-3. 소근육 발달 (8문항) |
| 사회성 또는 또래 상호작용 어려움이 의심된다 | 04-4. 사회성 및 상호작용 (8문항) |
| 인지발달 또는 학습준비 어려움이 의심된다 | 04-5. 인지발달 및 학습준비 (8문항) |
| 감각 예민함이 의심된다 | 04-6. 감각 예민함 (8문항) |
| 감각 둔감함 또는 감각추구 행동이 의심된다 | 04-7. 감각 둔감함 및 감각추구 (8문항) |
| 주의집중·감정조절·행동조절 어려움이 동반된다 | **대응 상세 섹션 없음** → 일반 플래그로 기록만 |
| 일상·기관생활·또래관계·학습활동 중 실제 불편이 발생 | **대응 상세 섹션 없음** → 일반 플래그로 기록만 |

- 규칙:
  - 03에서 "예"로 체크된 영역의 상세 섹션만 04 영역에 동적으로 노출한다.
  - 마지막 2개 항목은 매핑되는 상세 섹션이 없으므로 분기에 영향을 주지 않고 기록(02·03 응답으로 보존)만 한다.
  - 03에서 "예"가 하나도 없으면 04 상세 섹션은 노출되지 않으며, 곧바로 검토 화면으로 진행할 수 있다.
- 보조 옵션(권장): 검토 화면 또는 04 영역 상단에 **"표시되지 않은 영역도 직접 작성하기"** 토글을 제공해, 보호자가 원하면 미선별 영역도 펼쳐 작성할 수 있게 한다. (선택 구현 — §11)

### FR-6. 입력 검증
- 필수 항목 미입력 시 다음 단계 진행/제출 차단 및 인라인 오류 안내.
- 예/아니요 문항은 노출된 항목에 대해 응답 필수(미응답 방지). 단, 분기로 숨겨진 섹션은 검증 대상에서 제외.
- 연락처·날짜 형식 검증.

### FR-7. 검토/요약 화면
- 제출 전 입력값 전체를 확인하고 섹션으로 되돌아가 수정할 수 있다.
- "예"로 응답한 항목을 영역별로 요약 표시(상담사 빠른 파악용).
- ※ 자동 점수화/판정은 하지 않는다. 본 문서는 "검사 방향 설정 참고자료"이며 진단 도구가 아니다.

### FR-8. PDF 생성/출력
- 작성 결과를 **종이 양식과 동등한 레이아웃**의 PDF로 생성한다.
- PDF는 노출/응답된 항목을 모두 포함하며, 체크 상태(예/아니요)가 명확히 표기된다.
- 분기로 작성하지 않은 영역은 PDF에 "해당 없음/미선별"로 표기할지, 아예 생략할지 결정 필요(§11).
- 기능: PDF 다운로드, 브라우저 인쇄(프린터 연결 시).

### FR-9. 카카오톡 공유
- 작성 결과를 카카오톡으로 공유할 수 있다.
- 기술적 제약(검토 필요):
  - 카카오 공유 SDK는 기본적으로 **파일(PDF) 직접 전송이 아니라 링크/메시지 공유**다. PDF 파일 자체를 카톡으로 보내려면 (a) PDF를 임시 저장소에 업로드 후 **다운로드 링크를 카톡으로 공유**하거나, (b) 모바일 브라우저의 **Web Share API(`navigator.share`, 파일 첨부)**를 사용해야 한다.
  - "DB 영속 저장 없음" 방침과 (a)의 임시 업로드는 충돌 가능 → 보관 기간·자동 삭제 정책 또는 (b) 방식 채택 여부 결정 필요(§11).
- 공유 대상(보호자 본인 / 담당 상담사)도 결정 필요(§11).

### FR-10. 현장 기기 초기화(개인정보 보호)
- 공유/저장 완료 후 "처음으로/새 작성" 동작 시 입력값을 **완전히 삭제**한다.
- 일정 시간 미조작 시 자동 초기화(세션 타임아웃) 고려.
- 브라우저 로컬에 개인정보가 잔존하지 않도록 한다(자동 완성/캐시 관리).

## 7. 비기능 요구사항 (Non-Functional)

- **대상 화면 / 모바일 퍼스트**: **모바일 퍼스트**로 설계한다. 지원 대상은 **모바일과 태블릿** 화면에 한정하며, 데스크톱 풀사이즈(와이드) 레이아웃은 지원 대상이 아니다. 태블릿에서도 콘텐츠 폭을 모바일 기준으로 제한(중앙 정렬, 최대 폭 제한)하여 작성 흐름을 일관되게 유지한다.
- **모바일 UX**: 한 손 조작, 큰 터치 타깃(최소 44×44pt), 예/아니요는 토글/세그먼트 형태로 한 번에 응답.
- **접근성**: 충분한 명도 대비, 글자 크기 확대 대응, 라벨-입력 연결(스크린리더), 색에만 의존하지 않는 상태 표시.
- **성능**: 저사양 기기·느린 네트워크에서도 빠른 로드. 가능하면 오프라인 친화(작성 중 네트워크 끊김에도 입력 보존).
- **개인정보/보안**: 아동·보호자 민감정보 취급. 전송 시 HTTPS, 불필요한 서버 저장 지양, 비치 기기 잔존 데이터 제거. 개인정보 수집·이용 안내(동의 문구) 노출 여부 검토(§11).
- **국제화**: 1차 한국어 전용.
- **브라우저 지원**: 모바일 Safari(iOS), Chrome(Android) 최신 2개 버전 기준.

## 8. 데이터 모델 (폼 스키마 개요)

서버 영속 저장은 1차 범위 외지만, PDF 생성·공유를 위해 클라이언트 내 데이터 구조가 필요하다.

```
ConsultationForm {
  basicInfo: {
    childName, gender, birthDate,
    ageYears, ageMonths,        // birthDate 기반 자동 계산
    guardianName, guardianPhone
  }
  section02_basicCheck: { [questionId]: 'yes' | 'no' }      // 6문항
  section03_screening: { [domainId]: 'yes' | 'no' }         // 9문항 (분기 트리거)
  section04_details: {
    language?:        { [questionId]: 'yes' | 'no' }         // 04-1
    grossMotor?:      { [questionId]: 'yes' | 'no' }         // 04-2
    fineMotor?:       { [questionId]: 'yes' | 'no' }         // 04-3
    social?:          { [questionId]: 'yes' | 'no' }         // 04-4
    cognitive?:       { [questionId]: 'yes' | 'no' }         // 04-5
    sensorySensitive?:{ [questionId]: 'yes' | 'no' }         // 04-6
    sensorySeeking?:  { [questionId]: 'yes' | 'no' }         // 04-7
  }
  meta: { startedAt, completedAt, version }
}
```

- 문항 정의는 코드 상수/JSON 스키마로 분리하여 원본 문서와 1:1 대응·유지보수 용이하게 한다.
